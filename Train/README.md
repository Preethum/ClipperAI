# ClipperAI - Initial YOLO Training & Active Learning Pipeline

This directory contains the entire, end-to-end pipeline for extracting frames from gameplay videos, filtering them, intelligently annotating them using Active Learning, and training custom YOLO object detection models.

## Prerequisites
* Python 3.11+
* FFmpeg (Extracting frames)
* LM Studio (LLM Vision filtering)
* Label Studio (Manual annotation)
* PyTorch & Ultralytics (YOLO Training)
* PyQt6 & OpenCV (Interactive Active Learning GUI)

Make sure you've installed the necessary libraries in your virtual environment:
```cmd
pip install ultralytics torch torchvision torchaudio opencv-python PyQt6 requests tqdm
```

---

## 🚀 The Pipeline Workflow

### Step 1: Extract Frames
**`01_extract_frames.py`**
Takes a raw gameplay video (like Apex Legends or Overwatch) and extracts frames at a set interval (default 1 FPS) so the AI has images to look at.
* **Usage:** `python 01_extract_frames.py`
* **Input:** Raw `.mp4` video file.
* **Output:** Saves `.jpg` images into the `extracted_frames/` directory.

### Step 2 (Optional): Vision LLM Filtering
**`03_filter_frames.py`**
Not every frame is useful (e.g. menus, loading screens). This script uses a local Vision LLM via LM Studio to scan every frame and delete ones that don't contain gameplay UI or killfeeds.
* **Usage:** `python 03_filter_frames.py`
* **Input:** `extracted_frames/` directory.
* **Output:** Moves good frames to `import_to_label_studio/` and deletes useless ones.
* *Note: Requires LM Studio to be running a vision model on `localhost:1234`.*

### Step 3 (Crucial): Active Learning Scanner (GUI)
**`03b_auto_annotate.py`**
The crown jewel of the pipeline. Instead of manually looking at thousands of frames, this tool uses your *currently trained* YOLO model to scan a raw video and find **only the frames that confuse it**. 

It opens a desktop GUI where you can watch the video, and it will automatically pause when the AI isn't sure about a detection (e.g., between 20% and 75% confidence). You can then explicitly save that "hard" frame and the AI's best-guess labels directly into the Label Studio folder for manual correction.

* **Usage:** `python 03b_auto_annotate.py --video apex.mp4 --model runs/detect/train/weights/best.pt`
* **Output:** Saves pristine frames and `.txt` labels to `import_to_label_studio/`.

### Step 4: Manual Annotation
**`04_launch_label_studio.bat`**
Launches your local Label Studio instance pre-configured to handle massive local directories without crashing.
* **Usage:** `.\04_launch_label_studio.bat`
1. Create a "Local Storage" source pointing to `import_to_label_studio/`.
2. Sync the storage.
3. Review the "Uncertain Detections" generated by Step 3.
4. Correct the bounding boxes.
5. Export the finished dataset in **YOLO format** to the `YOLO_Data/` directory. 
   *(Note: Extract the downloaded zip so `classes.txt`, `images/`, and `labels/` are directly inside `YOLO_Data/`)*

### Step 5: Prepare Dataset for YOLO
**`05_prepare_yolo.py`**
YOLO requires a very specific `dataset.yaml` file to tell it where images are and what classes exist. This script safely splits up your annotated Label Studio export (`YOLO_Data/`) into 80% Training and 20% Validation folders, reads your `classes.txt`, and generates a perfect `dataset.yaml` file with absolute paths.
* **Usage:** `python 05_prepare_yolo.py`
* **Input:** `YOLO_Data/` directory (from Label Studio export).
* **Output:** `yolo_dataset/` directory and `yolo_dataset/dataset.yaml`.

### Step 6: Train the Model
**`06_train_yolo.py`**
Finally, feed the prepared dataset into the YOLO engine. It automatically detects and uses your NVIDIA GPU (CUDA) to train a new model.
* **Usage:** `python 06_train_yolo.py`
* **Input:** `yolo_dataset/dataset.yaml`.
* **Output:** All weights (`best.pt`), metrics, and charts are saved to `runs/detect/train/`.

### Step 7: Annotate a Full Video
**`07_predict_video.py`**
Takes your newly trained YOLO model and paints beautiful bounding boxes onto every frame of an entire raw video file, converting it into an annotated demo. Features a live progress bar.
* **Usage:** `python 07_predict_video.py --video apex.mp4 --model runs/detect/train/weights/best.pt --output annotated_apex.mp4`
* **Input:** Raw `.mp4` video file and your trained `.pt` model file.
* **Output:** A brand new `.mp4` video showing your AI in action.

## The Active Learning Loop (How to improve forever)
1. You trained an initial model (`runs/detect/train/weights/best.pt`).
2. You feed that model into the **Active Learning Scanner** (`03b`) with a new video.
3. The scanner finds 50 new "hard" situations the model failed on.
4. You correct those 50 situations in **Label Studio** (`04`).
5. You export the new, combined dataset and run **Prepare** (`05`).
6. You **Train** (`06`) a new model that is now vastly smarter than the old one.
7. Repeat!

## 📋 Quick-Run Command Sheet

Follow these steps sequentially to build and train your model.

### 🎬 Step 1: Extract Frames
```cmd
python 01_extract_frames.py
```
*Extracts frames from raw stream video directly into `extracted_frames/`.*

### 👁️ Step 2: OCR/Vision filter (Optional)
```cmd
python 03_filter_frames.py
```
*Intelligently filters frames via Vision LLMs to `import_to_label_studio/`.*

### 🔍 Step 3: Active Learning Auto-Annotation
```cmd
python 03b_auto_annotate.py --video "path/to/gameplay.mp4" --model "runs/detect/train/weights/best.pt"
```
*Scans video dynamically and locks on edge-confidence frames (saves to Label Studio directory).*

### 📝 Step 4: Manual Connection (Label Studio)
```cmd
.\04_launch_label_studio.bat
```
*Starts local instances for boxing; export outputs in YOLO format into `YOLO_Data/`.*

### 🏗️ Step 5: Prepare Dataset Subsets
```cmd
python 05_prepare_yolo.py
```
*Splits labels into structured 80/20 train/val partitions in `yolo_dataset/`.*

### 🚂 Step 6: Train Model
```cmd
python 06_train_yolo.py
```
*Kicks off YOLO training process reading `dataset.yaml` with automatic GPU triggers.*

### 🎥 Step 7: Predict Demo Video
```cmd
python 07_predict_video.py --video "apex.mp4" --model "runs/detect/train/weights/best.pt" --output "annotated.mp4"
```
*Renders boundary boxes onto full feeds showing your new items in action.*
